# Лабораторная работа №2


## Допущения
- Терминал сделан под Windows
- Флаги опций функции ставятся в любом месте после неё
- Ввод организован с помощью `input` и `while True`

Парсер разбивает строку на 3 части и возвращает словарь вида  
{"func": "название команды",  
"options":[опции],  
"args":[аргументы]}  
После чего вызывается функция выполняющая нужную команду

Для выхода из терминала надо ввести `quit` или `exit`
## Команды
### 1. touch и mkdir
```shell
~$ touch file
Создает файл с названием file, если его не существует

~$ mkdir dir
Создает директорию с назвнием dir, если её не существует, 
иначе выдаёт ошибку
```
### 2. ls
```shell
~$ ls [-l] [path]
Отображает содержимое директории, указанной в path 
(если path не указан, то содержимое текущей директории)

~$ ls -l
rwxrwxrwx 0 2025-11-04 16:16:26 direct
rw-rw-rw- 0 2025-11-04 16:16:34 file
rw-rw-rw- 0 2025-11-04 16:16:17 file.zip
rwxrwxrwx 0 2025-11-04 14:59:11 n
rw-rw-rw- 49 2025-11-04 14:58:58 new
rw-rw-rw- 49 2025-11-04 14:22:08 'new test'
```
### 3. cd
```shell
~$ cd path
Переход в каталог path 
(.. для перехода на уровень выше, ~ для перехода в домашнюю директорию)

~$ cd dir
~\dir$ cd ..
~$
```
### 4. cat
```shell
~$ cat file
Выводит текст файла file, если он в формате utf-8, иначе выдаёт ошибку
```
### 5. cp
Для копирования рекурсивно применяется функция `shutil.copy()`
```shell
~$ cp [-r] path dest
Копирует path в dest
Для копирования директории нужно указать -r
```
### 6. mv
```shell
~$ mv path dest
Перемещает path в dest, если dest существует, 
иначе переименовывает path в dest
Если path не сущуествует выдаёт ошибку
```
### 7. rm
```shell
~$ rm [-r] path
Удаляет path, если это директория, то сначала запрашивает разрешение 
Если path - директория и флаг -r не указан, то выдаёт ошибку
```
### 8. zip, unzip, tar, untar
```shell
~$ zip path dest[.zip]
Создаёт zip-архив в dest из path

~$ unzip path
Распаковывает архив path в текущую директорию
Если формат path не zip, выдаёт ошибку

~$ tar path dest[.tar.gz]
Создаёт tar.gz-архив в dest из path

~$ untar path
Распаковывает архив path в текущую директорию
Если формат path не tar.gz, выдаёт ошибку
```
### 9. grep
Поиск организуется с помощью `os.walk()` и применения регулярного выражения построчно для каждого файла
```shell
~$ grep [-r] [-i] pattern path
Ищет все совпадения pattern в path, -i - без учёта регистра
Выводит имя файла номер строки, в которой найдено совпадение и саму 
строку с совпадениями, выделенными красным цветом
Если path - директория и флаг -r не указан, то выдаёт ошибку

~$ grep hI dir -ri
dir\new на строке 1:`hi` oppsfd[p
dir\new на строке 2:jfhwoh`hi`
dir\new на строке 3:fi auhwuoifhwoe`hi`f`hi`
dir\dir\re на строке 1:hihfwophfherpgrte`hi`hweo
dir\dir\re на строке 3:fhgwguhwei`hi`ufhwi`hi`er`hi`wfrw 
(`` - выделено красным цветом)
```
